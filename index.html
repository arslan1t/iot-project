Why This Project Matters

Understanding the real-world challenges and the innovation behind our assistive navigation system

The Problem

Navigation tools for visually impaired people are often bulky, uncomfortable, and draw unwanted attention. Traditional white canes cannot detect elevated obstacles, and many assistive devices feel stigmatizing.

Our Solution

A discreet, ergonomic system integrated directly into footwear. Smart shoes scan the environment, detect obstacles, and notify users through intuitive haptic feedback — all without drawing attention.

How the System Works

Sensors & Audio Interaction

The system continuously collects data from multiple sensors installed in the shoe. Ultrasonic sensors detect obstacles in front of the user. Each sensor event is mapped to a specific sound pattern so the user can understand the environment without visual feedback.

Slip detection is handled in the dedicated slip-detection module.

Central Control Unit

Arduino Uno acts as the central controller of the system. It processes all sensor inputs in real time and decides which feedback method should be activated. A priority-based logic is implemented so that emergency events have the highest priority, and navigation feedback runs only when no critical events are active. Slip alerts come from the dedicated slip-detection block.

Safety & Feedback Modules

The system uses multiple feedback channels to ensure user safety. Audio signals are the primary feedback method, while vibration and visual indicators provide additional system status information. Each feedback channel has a dedicated role and does not conflict with higher-priority alerts.

My Contribution — Firmware & Safety Logic

I was responsible for designing and implementing the core firmware logic running on the Arduino Uno, with a focus on safety prioritization, non-blocking execution, and intuitive feedback for blind users.

Non-Blocking Firmware Architecture

I implemented the main control loop using millis()-based scheduling instead of blocking delays. This allows ultrasonic sensing, pressure monitoring, emergency handling, and feedback generation to run concurrently without freezing the system. 

// Non-blocking sensor update
if (now - lastSensorMillis >= SENSOR_INTERVAL_MS) {
  lastSensorMillis = now;
  readAllSensors();
}

This approach ensures predictable timing and stable behavior even when multiple alerts are active.

Priority-Based Safety Logic

I designed and implemented a strict priority hierarchy to prevent conflicting feedback. Critical events always override lower-priority signals. 

// Priority order:
// 1. Emergency
// 2. Slip detection
// 3. Obstacle navigation
if (emergencyActive) {
  playEmergencySound();
} else if (pressureAlertActive) {
  playSlipAlert();
} else {
  playUltrasonicSound(closestDistance);
}

This guarantees that the user never receives mixed or misleading signals.

Slip Detection Logic (HX711)

I implemented slip detection using a pressure sensor connected via the HX711 module. Instead of a single threshold, the firmware analyzes pressure trends and timing between steps. 

// Slip detection conditions
if (pressure < MIN_PRESSURE) slip = true;
if ((lastPressure - pressure) > DROP_THRESHOLD) slip = true;
if (millis() - lastStepTime > STEP_TIMEOUT) slip = true;

This logic helps detect slippery or unstable surfaces before a fall occurs.

Adaptive Audio & Haptic Feedback

I implemented distance-based audio feedback where the beep interval decreases as obstacles get closer, creating an intuitive spatial cue. Vibration feedback is triggered for near obstacles and critical states. 

// Beep rate increases as distance decreases
interval = map(distance,
  WARN_DISTANCE,
  CRITICAL_DISTANCE,
  800,
  150
);

This allows blind users to estimate distance without verbal instructions.

Adaptive Sound Feedback

Audio feedback is the primary communication channel of the system. Different sound patterns are used to represent navigation, surface danger, and emergency states without requiring visual attention.

Event Sound Pattern Purpose
Obstacle Beep speed increases as distance decreases Spatial navigation
Slip High-frequency double beep Surface danger warning
Emergency Low-frequency repeating tone Immediate critical alert

Obstacle Navigation

Beep frequency increases as the distance to an obstacle decreases, providing intuitive spatial awareness during walking.

Slip Detection Warning

A sudden pressure drop or loss of stable contact triggers a high-frequency double beep to warn about a slippery surface.

Emergency Alert

Emergency mode produces a slow, repeating low-frequency tone. It is manually triggered and immediately overrides all other alerts.
