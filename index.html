Why This Project Matters

Understanding the real-world challenges and the innovation behind our assistive navigation system

The Problem

Navigation tools for visually impaired people are often bulky, uncomfortable, and draw unwanted attention. Traditional white canes cannot detect elevated obstacles, and many assistive devices feel stigmatizing.

Our Solution

A discreet, ergonomic system integrated directly into footwear. Smart shoes scan the environment, detect obstacles, and notify users through intuitive haptic feedback — all without drawing attention.

How the System Works

Obstacle Detection

Ultrasonic and LIDAR sensors continuously scan 15–150 cm in front of the user, detecting ground and upper-level obstacles.

Adaptive Signal System

ESP32 processes sensor data in real-time, adjusting vibration feedback intensity as obstacles get closer.

GPS Emergency Tracking

GPS NEO-6M enables emergency location broadcasting with ±5m accuracy and optional WiFi/Bluetooth transmission.

My Contribution: Firmware & Safety Logic

This section highlights my personal contribution to the project, focusing on embedded logic, sensor behavior modeling, and real-time safety prioritization.

Embedded Firmware Design

I developed the core firmware logic responsible for processing sensor data in real time. The system continuously analyzes ultrasonic distance measurements and pressure sensor readings without blocking delays, ensuring stable and responsive behavior.

Priority-Based Safety System

A strict priority hierarchy was implemented to prevent conflicting feedback. Emergency alerts override all signals, slip detection overrides navigation sounds, and obstacle feedback is enabled only when no higher-priority event is active.

Slip Detection Using Pressure Sensor

Detection of unsafe or slippery surfaces using dynamic pressure analysis

How the Sensor Works

The pressure sensor continuously measures foot-ground contact force. Instead of using a single threshold, the firmware analyzes pressure trends and time-based behavior to detect abnormal patterns.

Slip Detection Conditions

  • Pressure drops below a safe minimum value
  • Sudden pressure decrease between steps
  • Loss of stable contact for a defined time window

When a slip is detected, the system immediately disables navigation sounds and triggers a dedicated warning pattern to alert the user.

Adaptive Sound Feedback Logic

Intuitive audio feedback designed for navigation without visual cues

Event Type Sound Pattern Purpose
Emergency Low-frequency slow repeating tone Immediate critical alert
Slip Detection High-frequency double beep Surface instability warning
Obstacle Navigation Beep speed increases as distance decreases Spatial distance awareness

For obstacle detection, the closer the obstacle is, the faster the sound interval becomes, creating an intuitive audio representation of distance.

Why This Firmware Logic Matters

Engineering decisions behind the safety and usability of the system

User Safety

By separating alerts into clear priority levels, the system avoids overwhelming the user with multiple signals and ensures that critical events are always clearly communicated.

Real-Time Reliability

All logic is implemented using non-blocking timers instead of delays, allowing stable operation and fast response times even under multiple simultaneous sensor inputs.
