Smart Shoes for Safe, Guided Walking

A university-built assistive navigation system that uses an Arduino Uno, ultrasonic sensors, and a pressure-based slip detector to support safe movement for visually impaired users.

Explore Project ↓ Impact Open PDF ↗

Why This Project Matters

Understanding the real-world challenges and the innovation behind our assistive navigation system

The Problem

Navigation tools for visually impaired people are often bulky, uncomfortable, and draw unwanted attention. Traditional white canes cannot detect elevated obstacles, and many assistive devices feel stigmatizing.

Our Solution

A discreet, ergonomic system integrated directly into footwear. Smart shoes scan the environment, detect obstacles, and notify users through intuitive haptic feedback — all without drawing attention.

How the System Works

Sensors & Audio Interaction

The system continuously collects data from multiple sensors installed in the shoe. Ultrasonic sensors detect obstacles in front of the user. Each sensor event is mapped to a specific sound pattern so the user can understand the environment without visual feedback.

Slip detection is handled in the dedicated slip-detection module.

Central Control Unit

Arduino Uno acts as the central controller of the system. It processes all sensor inputs in real time and decides which feedback method should be activated. A priority-based logic is implemented so that emergency events have the highest priority, and navigation feedback runs only when no critical events are active. Slip alerts come from the dedicated slip-detection block.

Safety & Feedback Modules

The system uses multiple feedback channels to ensure user safety. Audio signals are the primary feedback method, while vibration and visual indicators provide additional system status information. Each feedback channel has a dedicated role and does not conflict with higher-priority alerts.

Technical Workflow

Environmental Sensing

Three HC-SR04 modules poll left/center/right distances in a staggered schedule to avoid crosstalk.

Data Processing

Arduino Uno runs a non-blocking loop with time-sliced tasks, applying distance thresholds and debounce windows.

Feedback Generation

Buzzer, vibration motor, RGB LED, and LCD work together to convey obstacle distance and alerts.

Pressure-based slip detection is described in the dedicated slip-detection section.

Detailed Technical Specifications

Complete hardware and software specifications of the smart-shoe navigation system

Processing Unit

  • Arduino Uno (ATmega328P)
  • 16 MHz loop with non-blocking timing (millis)
  • Priority scheduling for safety alerts
  • Serial logging + mock GPS/voice output
  • Debounce for emergency button (long press)
  • LCD 16x2 for status and debug text

Sensor Array

  • 3× HC-SR04 (left / center / right)
  • Staggered triggering to reduce crosstalk
  • HX711 + pressure pad for slip tendency
  • Typical useful range: ~20–120 cm indoors
  • Emergency button (long press)
  • Serial mock GPS/voice messages

Feedback Systems

  • Piezo buzzer with distance-based beep rate
  • Single vibration motor for close obstacles
  • RGB LED color codes per priority
  • LCD 16x2 textual cues
  • Serial prompts for simulated voice/GPS
  • Manual emergency alert pattern

Adaptive Sound Feedback Logic

Intuitive audio feedback designed for navigation without visual cues

Event Type Sound Pattern Purpose
Emergency Low-frequency slow repeating tone Immediate critical alert
Slip Detection High-frequency double beep Surface instability warning
Obstacle Navigation Beep speed increases as distance decreases Spatial distance awareness

For obstacle detection, the closer the obstacle is, the faster the sound interval becomes, creating an intuitive audio representation of distance.

Why This Firmware Logic Matters

Engineering decisions behind the safety and usability of the system

User Safety

By separating alerts into clear priority levels, the system avoids overwhelming the user with multiple signals and ensures that critical events are always clearly communicated.

Real-Time Reliability

All logic is implemented using non-blocking timers instead of delays, allowing stable operation and fast response times even under multiple simultaneous sensor inputs.

Slip Detection Using Pressure Sensor

Detection Logic

HX711-based pressure trend analysis flags unsafe states:

  • Pressure below safe minimum
  • Sudden pressure drop between steps
  • Loss of stable contact over time

Why It Matters

Slippery surfaces raise fall risk; early pressure-based detection reduces that risk in the prototype.
